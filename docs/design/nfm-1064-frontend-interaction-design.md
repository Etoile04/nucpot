## 1. 信息架构与页面地图

### 1.2 用户旅程图

#### 旅程A: 研究人员查找材料属性
```
首页 → 搜索 → 材料详情 → 属性数据 → 关联文献
                ↓
           浏览 → 筛选 → 材料详情 → 对比势函数
```

#### 旅程B: 工程师下载势函数
```
首页 → 浏览(势函数) → 势函数详情 → 下载
                ↓
           搜索 → 筛选类型/元素 → 对比多个势函数 → 下载
```

#### 旅程C: 数据贡献者上传数据
```
首页 → 登录 → 上传 → 选择类型 → 分步填写 → 提交 → 等待审核
```

## 2. 设计系统约束

### 2.1 必须使用的现有 Token

> 硬性规则: 所有设计值必须引用现有 CSS Custom Properties，禁止引入硬编码值。

#### 颜色系统

| 用途 | Token | 值 |
|------|-------|-----|
| 页面背景 | `--color-surface` | `#1f2937` |
| 主文本 | `--color-text` | `#f9fafb` |
| 次要文本 | `--color-text-secondary` | `#9ca3af` |
| 强调色/链接 | `--color-accent` | `#60a5fa` |
| 边框 | `--color-border` | `#374151` |
| 表单标签 | `--form-label-color` | `#e5e7eb` |
| 输入框背景 | `--form-input-bg` | `#111827` |
| 输入框边框 | `--form-input-border` | `#374151` |
| 输入框聚焦 | `--form-input-focus-border` | `#3b82f6` |

| EAM | `blue-400` | 标签/卡片标识 |

#### 字体

| Token | 值 | 用途 |
|-------|-----|------|

### 2.2 组件库优先级

1. **Ant Design 5** — 管理后台、表单、表格、模态框、消息提示
2. **Tailwind CSS 4** — 公开页面布局、间距、响应式
3. **ECharts 5** — 数据可视化（使用 `nfmDarkTheme`）
4. **自定义组件** — 仅当 Ant Design 无法满足时新增

### 2.3 禁止引入

- 任何新的一行 CSS 不使用现有 token
- 新的 NPM UI 组件库
- 硬编码像素值（必须引用 token 或 Tailwind spacing scale）

#### 交互规格

| 元素 | 组件 | Token/Props | 交互行为 |
|------|------|-------------|----------|
| 筛选面板 | 自定义 Checkbox 组 | 宽度 `w-64`, 背景 `--color-surface` | 折叠/展开各分组，选中项实时过滤 |
| 视图切换 | `Antd Segmented` | 两个图标选项 (list/grid) | 切换时结果区无刷新过渡 |
| 结果卡片 | `<PotentialCard>` (已有) | 类型色彩标签按 2.1 编码 | 点击进入详情页 |

| 搜索过滤 | `Antd Input` | 内联在结果区顶部 | 输入 debounce 300ms 后过滤 |

#### 筛选面板行为

- **展开/折叠:** 各筛选分组默认展开，点击组标题可折叠
- **清除筛选:** 面板顶部「清除所有筛选」按钮
- **计数徽标:** 每个筛选项旁显示匹配数量 (如 `FCC (23)`)
- **URL 同步:** 所有筛选状态同步到 URL search params (`?category=metal&structure=fcc`)
- **移动端:** 筛选面板变为底部抽屉 (`Antd Drawer`)

#### 交互规格

| 元素 | 组件 | Token/Props | 交互行为 |
|------|------|-------------|----------|
| 搜索框 | `Antd Input.Search` | `size="large"`, `allowClear` | 回车搜索，空输入显示提示 |
| 高级筛选面板 | `Antd Collapse` | 默认收起，背景 `--color-surface` | 展开后显示 3 列筛选器 |
| 属性类别 | `Antd Cascader` | 三级联动: 类别-属性-条件 | 选中后出现数值范围输入 |
| 温度/压力范围 | `Antd InputNumber` 双端 | `--form-input-bg` 背景 | 范围验证 (min < max) |
| 结果卡片 | 自定义 `<SearchResultCard>` | 左侧图标 + 内容 | 点击跳转对应详情页 |
| 结果排序 | Tab 切换 | 相关性 / 最新 / 属性数量 | 切换排序方式 |

#### 渐进式披露策略

1. **首次访问:** 仅显示搜索框 + 最近热门搜索标签
2. **搜索后:** 显示高级筛选面板（默认收起）+ 结果
3. **空结果:** 显示推荐搜索建议 + 宽泛条件提示
4. **大量结果:** 显示 facet 聚合（按分类/属性类型分布）

| 文献详情 | `Antd Descriptions` | 两栏布局 | 展示提取的实体和关联材料 |

## 4. 响应式设计策略

### 4.1 断点定义

| 断点 | 宽度 | 设备 | 布局策略 |
|------|------|------|----------|
| `xs` | < 640px | 手机竖屏 | 单列，底部导航抽屉 |
| `sm` | 640-767px | 手机横屏 | 单列，侧边栏抽屉 |
| `md` | 768-1023px | 平板 | 双列，侧边栏可折叠 |
| `lg` | 1024-1439px | 桌面 | 标准三栏布局 |
| `xl` | >= 1440px | 大桌面 | 加宽内容区 |

### 4.2 关键页面响应式行为

#### 导航栏

| 断点 | 行为 |
|------|------|
| >= `md` | 水平导航栏，所有菜单可见 |
| < `md` | 汉堡菜单图标，点击展开 `Antd Drawer` 侧滑 |

#### 浏览页 (/browse)

| 断点 | 筛选面板 | 结果区 |
|------|----------|--------|
| >= `lg` | 左侧固定侧栏 `w-64` | 右侧自适应 |
| `md` | 可折叠侧栏 (默认收起) | 全宽 |
| < `md` | 底部抽屉 (点击筛选按钮唤出) | 全宽, Card 单列 |

#### 高级搜索 (/search)

| 断点 | 筛选面板布局 |
|------|-------------|
| >= `lg` | 三列并排 |
| `md` | 三列自适应 |
| < `md` | 垂直堆叠 |

#### 详情页 (/material/[id], /potential/[id])

| 断点 | 布局 |
|------|------|
| >= `md` | 信息卡片 + 成分表并排 |
| < `md` | 垂直堆叠 |
| >= `lg` | 属性表格全宽，固定首列 |
| < `lg` | 属性表格可横向滚动，或改为卡片列表 |

#### 对比页 (/compare)

| 断点 | 行为 |
|------|------|
| >= `md` | 对比表格横向并排 |
| < `md` | 对比表格改为垂直堆叠卡片模式 |

### 4.3 触控适配

- 所有可交互元素最小触控区域: **44x44px** (WCAG 2.2)
- 表格行间距增加至 `padding-y: 12px` (移动端)
- 筛选面板使用 Radio Group / Checkbox (非 hover 下拉)
- 搜索框输入后显示「搜索」按钮 (不依赖回车)

## 5. 交互状态规范

### 5.1 加载状态

| 场景 | 组件 | 行为 |
|------|------|------|
| 首次加载页面 | `Antd Spin` / `Skeleton` | 内容区显示 Skeleton 骨架屏 |
| 表格数据加载 | `Antd Table` `loading` prop | 表格内显示 loading 骨架 |
| 图表数据加载 | ECharts `loading` 配置 | 图表区显示加载动画 |
| API 请求 | TanStack Query `isLoading` | 按钮显示 loading + 禁用 |

### 5.2 空状态

| 场景 | 显示内容 | 操作引导 |
|------|----------|----------|
| 搜索无结果 | 空状态插图 + "未找到匹配结果" | 「清除筛选」+ 「查看全部数据」 |
| 浏览无匹配 | "当前筛选无匹配" | 「调整筛选条件」 |
| 详情页数据缺失 | "暂无数据" + 灰色虚线框 | 「贡献数据」按钮 |
| 用户未登录 | 登录提示 | 「去登录」按钮 |

### 5.3 错误状态

| 场景 | 组件 | 行为 |
|------|------|------|
| 网络错误 | `Antd Result` `status="error"` | 显示重试按钮 |
| 401 未授权 | 全局 Auth 拦截 | Toast 提示 + 跳转 `/login` |
| 404 页面不存在 | `Antd Result` `status="404"` | 提供返回首页链接 |
| 表单验证失败 | `Antd Form` `validateFeedback` | 字段级错误提示 (红色) |
| API 业务错误 | `Antd message.error` | Toast 显示后端错误信息 |

### 5.4 成功状态

| 场景 | 组件 | 行为 |
|------|------|------|
| 数据上传成功 | `Antd Result` `status="success"` | 显示成功消息 + 「查看已上传」链接 |
| 反馈提交成功 | `Antd message.success` | Toast + 表单清空 |
| 审核操作 | `Antd message.success` | Toast + 列表自动刷新 |

## 6. 无障碍与国际化

### 6.1 无障碍要求

| 要求 | 实现 |
|------|------|
| 键盘导航 | 所有交互元素可通过 Tab 聚焦，Enter/Space 激活 |
| 屏幕阅读器 | 表格使用 `caption`，图标提供 `aria-label`，图片提供 `alt` |
| 颜色对比度 | 文本对比度 >= 4.5:1 (WCAG AA)，当前暗色主题需验证 |
| 焦点指示 | 使用 Tailwind `focus-visible:ring-2 ring-[var(--color-accent)]` |
| 减少动效 | 尊重 `prefers-reduced-motion`，禁用非必要动画 |
| 语义化 HTML | 使用 `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>` |

### 6.2 CJK 排版规范

| 规则 | 说明 |
|------|------|
| 行高 | 中文内容 `line-height: 1.75`，中英混排 `1.6` |
| 字间距 | 中文正文不加 letter-spacing |
| 段间距 | 中文章节 `margin-bottom: 1.5em` |
| 数字对齐 | 表格数值列使用 `font-variant-numeric: tabular-nums` |
| 科学记数 | 材料分子式使用 `<sub>` 标签 (UO₂ → UO<sub>2</sub>) |

### 6.3 国际化策略

- **当前阶段:** 中文为主，英文为辅 (标签/工具提示)
- **Antd locale:** 已配置 `zh_CN`
- **文案管理:** 统一在 `src/i18n/` 或组件内 props 传入
- **单位显示:** 使用国际标准单位 (SI)，温度显示 K (开尔文)
