## 1. 核心辨析：协议 vs 实现

当有人说"X 不适合国内合规"时，先做拆解：

| 问题 | 答案模式 |
|------|----------|
| **API 协议**（如 S3 REST）本身是合规风险吗？ | 几乎从来不是。S3 API 是事实开放标准（Oracle v. Google 2021——API 不受版权保护）。国内所有主流云厂商（阿里云 OSS、腾讯云 COS、华为云 OBS）都 S3 兼容。银行、电信运营商大规模在用 S3 API。 |
| **具体实现**（如 MinIO Inc.）是合规风险吗？ | 可能。MinIO 是 AGPL v3（网络 copyleft）、美国公司、**不在信创目录**。Ceph RGW 是 LGPL（友好），已全面信创适配，是中国国企事实标准。 |

**切勿把两者混为一谈。** `StorageBackend` 抽象层 + boto3 的设计在任何情况下都正确——切换实现只需改环境变量。

### 1.1 合规真正约束的是什么（不是协议）

| 约束项 | 真实要求 | 与 S3 API 是否相关 |
|--------|----------|---------------------|
| 数据驻留 | 数据存于中国境内物理服务器 | 否——取决于部署位置 |
| 传输加密 | 需要 TLS；部分场景需 SM2/SM4 | 否——HTTPS 通用 |
| 存储加密 | 静态加密；部分需 SM4 | 否——应用层关注点 |
| 访问审计 | 完整访问日志，留存 ≥6 个月 | 否——网关/应用层关注点 |
| 密码合规 | 商用密码产品认证 | 部分——签名算法可用国密 |
| 供应链 | 软件自主可控、无后门 | 与具体实现相关，与协议无关 |

## 2. 信创（Xinchuang）目录

- **信创** = 信息技术应用创新，工信部主管
- 目标：构建自主可控的 IT 供应链，替代国外产品
- 四大领域：CPU（鲲鹏/海光/飞腾/龙芯）、OS（麒麟/统信 UOS/欧拉）、数据库（达梦/人大金仓/OceanBase）、中间件
- **信创目录要求国内注册公司**——国外开源软件（MinIO 等）无法进入
- 目录清单通常非公开（仅在采购系统中可见）

## 3. 等保 2.0（Dengbao）级别与存储要求

GB/T 22239-2019，对象存储归入"云计算安全扩展"：

| 级别 | 数据位置 | 加密 | 审计 | 适用对象 |
|------|----------|------|------|----------|
| 二级 | 国内 | 传输 | 基础日志 | 一般企业 |
| **三级** | **国内、明确驻留** | **存储+传输** | **完整审计** | **金融/医疗/政务关键系统** |
| 四级 | 国内、物理隔离 | 强加密 | 完整审计 | 关键基础设施 |
| 五级 | 专网 | 国密 | 受控 | 涉密系统 |

**核材料数据 → 至少三级基线。**

## 6. 涉密+信创全栈评估：升级模式

当确认"需要考虑涉密场景" + "采购方可能强制信创目录"时，问题从"用哪个存储后端"升级为"**整套技术栈能否通过涉密+信创审查**"。每一层都要评估：

| 层 | 当前（开发） | 涉密+信创要求 | 迁移难度 |
|----|--------------|---------------|----------|
| **硬件 CPU** | Apple Silicon | 鲲鹏 920（首选）/海光（x86 过渡）/飞腾 | 零代码改动 |
| **硬件 GPU** | 无 / Apple MLX | 昇腾 910B（~60–80 万/卡）/寒武纪思元/海光 DCU | 零代码（仅部署） |
| **OS** | macOS | 银河麒麟 V10 SP3 / openEuler 22.03 LTS | 零代码——Docker CE 两者均可 |
| **容器运行时** | Docker Desktop | Docker CE / Podman / iSulad | 零——compose 文件不变 |
| **对象存储** | MinIO（美国，AGPL） | **XSKY XEOS** / 杉岩 MOS / 华为 OceanStor（信创目录） | **零代码**——StorageBackend 抽象 |
| **数据库** | PostgreSQL 16 | **人大金仓 KingbaseES**（PG 分支，最易迁移）/GaussDB | 低——SQLAlchemy dialect 切换 |
| **Hash 算法** | SHA-256 | **SM3**（输出长度相同：64 hex 字符） | 极小——1–2 行 |
| **对称加密** | AES-256 | **SM4** | 极小 |
| **非对称加密** | RSA | **SM2** | 中——JWT/SSH |
| **TLS/SSL** | 标准 TLS | 国密 SSL（SM2 双证书 + Tengine/BabaSSL） | 中—高 |
| **S3 签名** | boto3 SigV4（HMAC-SHA256） | 厂商 boto3 分支（SM3）或在国密 TLS 下容忍 SigV4 | 技术上高，若厂商提供分支则低 |
| **PDF 解析** | PyMuPDF（AGPL） | pypdf / pdfplumber / MinerU（信创友好许可） | 低 |
| **网络暴露** | Cloudflare Tunnel | **彻底删除**——内部 nginx 反代 | 中 |
| **CI/CD** | GitHub Actions | Gitea Actions / GitLab CI（内部） | 中 |
| **镜像/包源** | Docker Hub / PyPI / npm | 内部 Harbor + Nexus / devpi | 中（仅部署） |
| **LLM 推理** | Ollama + 可能外部 API | Ollama + 国产模型（Qwen/GLM/Baichuan）+ 昇腾 GPU | 低（仅配置） |
| **外部 LLM API** | GLM/DeepSeek/Claude | **彻底删除**——100% 本地推理 | 中 |
| **审计/认证** | 无 | 等保三级 + 涉密保密测评 + 商密认证（GM/T 0028） | 6–12 个月流程 |

### 6.1 关键洞察：~80% 代码可存活

让迁移可承受的两项架构资产：

1. **StorageBackend Protocol**——存储 MinIO→XSKY 是 4 个环境变量
2. **SQLAlchemy ORM**——数据库 PG→KingbaseES 是连接串 + dialect

没有这两层抽象，迁移等于全部重写。
有了它们，**~80% 代码零改动**；剩下的是密码、配置与部署。

## 7. 国密（SM2/SM3/SM4）改造

### 7.1 Python 实现

| 算法 | 替代 | Python 实现 | 备注 |
|------|------|-------------|------|
| SM2 | RSA-2048+ | `pycryptodome`（C 扩展，推荐）/ `gmssl`（纯 Python，慢） | 商密产品认证需厂商封装，非裸 pycryptodome |
| SM3 | SHA-256 | Python 3.11+ `hashlib.new('sm3')` / `pycryptodome` | 输出 256 位 = 64 hex 字符——与 SHA-256 一致，无 schema 改动 |
| SM4 | AES-128/256 | `pycryptodome` / `gmssl` | ECB/CBC/GCM 模式 |

### 7.2 Hash 参数化是零成本前置项

**即便现在用 SHA-256，也要把它抽象到环境变量后**——这样切到 SM3 只是配置改动，不是代码改动。
SM3 输出长度 = SHA-256 输出长度，DB 列定义（VARCHAR(64)）不变。

### 7.3 架构抽象存活清单

设计一个未来可能需要涉密+信创适配的系统时，现在就做好这些：

| 准备 | 现在做的理由 | 工作量 |
|------|--------------|--------|
| Hash 算法参数化（sha256/SM3） | SM3 输出长度 = SHA-256，无 schema 改动 | 极小（1–2 处） |
| 加密算法抽象（AES/SM4） | 应用层加密，不绑定单一算法 | 低 |
| LLM 调用确保本地兜底 | 涉密网 = 无外部 API | 验证配置 |
| 网络入口支持内部模式（nginx） | 涉密 = 无 Cloudflare Tunnel | 低 |
| 通过 ORM 保证数据库可移植（SQLAlchemy） | PG→KingbaseES 是 dialect 切换，非重写 | 用了 SQLAlchemy 即已就绪 |

## 8. 涉密三条红线（BMB17-2006，不可妥协）

依据《涉及国家秘密的信息系统分级保护技术要求》：

1. **物理隔离**——无任何互联网连接。Cloudflare Tunnel 必须删除，所有外部 API 调用移除，模型文件离线预下载。
2. **国密算法**——SM3/SM4/SM2 全面替换，不得使用 SHA-256/AES/RSA。
3. **介质管控+密级标识**——存储 bucket 标注密级，所有读写审计，介质退役前物理销毁。

## 10. 关键陷阱：不要照单全收"S3 不合规"

当有人提出该顾虑时，应：

1. **承认底层顾虑有效**（数据安全很重要）
2. **拆解**："S3 API 协议" vs "MinIO 具体实现"——它们不同
3. **澄清**：真正的约束是数据驻留、加密、审计——不是协议
4. **举证**：国内云厂商全部使用 S3 API → 协议没问题
5. **再回应实现顾虑**：若信创/采购是硬要求，MinIO → Ceph RGW / XSKY，但抽象层让这成为环境变量切换

| 阶段 | 周期 | 目标 |
|------|------|------|
| 1：环境搭建 | 1–2 月 | 鲲鹏服务器 + 麒麟 OS + Docker CE + 内部 Harbor/Nexus |
| 2：存储+数据库迁移 | 2–3 周 | KingbaseES 替代 PG + XSKY 替代 MinIO（通过抽象层） |
| 3：国密改造 | 2–4 周 | SM2/SM3/SM4 全栈替换 + 国密 TLS |

| 6：认证 | 6–12 月 | 并行：等保 + 涉密 + 商密 |
