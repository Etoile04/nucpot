## 1. 项目概述

核燃料与材料物性数据库平台（NFM-DB）面向中国核能行业对高质量、可持续共享的材料物性数据的需求，构建从多模态数据智能提取、数据融合治理、领域知识约束ML到多目标优化的全链路平台。

### 核心价值主张

1. **数据汇聚**：整合分散在文献、报告、实验记录中的核材料物性数据
2. **AI驱动提取**：多模态（PDF/图片/表格）自动结构化提取
3. **知识图谱约束**：领域本体指导数据质量与ML训练
4. **智能设计推荐**：团簇/胶原子ML模型辅助新材料设计

### 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端 | Next.js 15 (App Router) + TypeScript + Ant Design 5 |
| 后端 | Python 3.12+ / FastAPI |
| 数据库 | PostgreSQL 16 + Alembic |
| ML | PyTorch + MACE/NequIP (消息传递势函数) |
| 部署 | Docker + GitHub Actions CI/CD |

## 3. 技术路线

```
多模态数据智能提取 → 数据融合治理 → 领域知识约束ML → 多目标优化 → 实验验证
       (Sprint 4)          (Sprint 5-6)       (Sprint 4-5)      (Sprint 6)     (Sprint 7)
```

### 路线设计原则

1. **先攻坚核心**：智能设计引擎作为最大创新点，Sprint 4即启动
2. **基础设施先行**：数据治理（Sprint 6前置到Sprint 5部分）保障ML数据质量
3. **演示驱动**：每阶段交付可演示成果，支持竞赛答辩可视化
4. **风险可控降级**：ML不达标时有明确回退方案（见§9）

## 6. 智能设计引擎：量化指标与验证基准

### 6.1 模型架构

采用**等变消息传递神经网络**（Equivariant Message Passing Neural Network），具体选型MACE或NequIP架构，适用于周期性体系的原子级势函数预测。

**为什么选择等变架构**：
- 天然满足旋转/平移等变性，物理先验强
- 在OEIS/ Materials Project等基准上MAE比传统MLP低40-60%
- 团簇体系（非周期性边界）与体相体系均可处理

### 6.2 训练数据

| 项目 | 规格 |
|------|------|
| 数据来源 | 核动力院实验数据 + DFT计算数据 + 文献提取数据 |
| 训练集规模 | **≥5000条**团簇构型（Sprint 4目标：2000条起） |
| 特征维度 | 原子种类(Z) + 坐标(r) + 团簇尺寸(N) + 电子态标签 |
| 数据划分 | 80%训练 / 10%验证 / 10%测试，按团簇尺寸分层抽样 |

### 6.3 验证基准（量化指标）

#### 6.3.1 目标指标

| 物理量 | DFT基准精度 | ML目标MAE | ML目标R² | 验证方法 |
|--------|------------|----------|---------|----------|
| **团簇形成能** (eV) | ±0.05 | **≤0.03 eV/atom** | **≥0.95** | 5-fold交叉验证 |
| **键长** (Å) | ±0.02 | **≤0.01 Å** | **≥0.98** | 留出集预测 |
| **结合能** (eV) | ±0.10 | **≤0.05 eV** | **≥0.93** | K-fold验证 |
| **HOMO-LUMO gap** (eV) | ±0.20 | **≤0.15 eV** | **≥0.90** | 统计检验 |

#### 6.3.2 基准对比

```
精度等级定义：
  Level 1 (PPT工程): R² < 0.80, 仅定性趋势正确 → ❌ 不可接受
  Level 2 (初赛可用): R² 0.80-0.90, 定量误差 < 2× DFT基准 → ⚠️ 勉强
  Level 3 (决赛推荐): R² ≥ 0.90, MAE ≤ DFT基准精度 → ✅ 目标
  Level 4 (发表级):   R² ≥ 0.95, MAE < DFT基准精度 → 🏆 超预期
```

#### 6.3.3 评估流程

1. **Sprint 4**：用2000条数据训练MACE-Small（~8层，256通道），5-fold CV
2. **Sprint 5**：扩展至5000条，切换MACE-Large（~12层，512通道）
3. **Sprint 6**：外部验证 — 预测3-5个核动力院未公开实验值，对比实测
4. **Sprint 7**：ablation study — 消融等变性/数据增强/特征工程

### 6.4 胶原子模型

胶原子（glue atom）模型将团簇分解为"胶原子"与"基本原子"两类：
- **胶原子**：维持团簇结构稳定性的关键原子
- **基本原子**：可通过替换/掺杂改变性能

ML预测目标：给定元素组合 → 预测最优胶原子位置与团簇形成能

## 9. 风险预案

### 9.1 风险矩阵

| # | 风险 | 概率 | 影响 | 当前缓解 |
|---|------|------|------|----------|
| R1 | ML精度不达标（R² < 0.85） | 高 | 致命 | 见Plan B |
| R2 | 核动力院实验数据未按时到位 | 中 | 高 | 公开数据集fallback |
| R3 | 竞赛时间压缩导致演示不完整 | 高 | 高 | 每Sprint交付可演示成果 |
| R4 | GPU资源不足 | 低 | 中 | 已有预算，备用T4方案 |

### 9.2 Plan B：ML精度不达标的降级策略

**触发条件**：Sprint 5结束时（07-29），ML模型R² < 0.85

**降级路径**（按严重程度递进）：

| 等级 | 条件 | 策略 | 答辩叙述 |
|------|------|------|----------|
| **Plan B-1** | R² 0.85-0.90 | 聚焦单一物理量（键长/形成能），放弃全面预测 | "我们选择最难预测的形成能作为突破口" |
| **Plan B-2** | R² 0.80-0.85 | 降模型复杂度（MACE-Small），扩大训练集 + 数据增强 | "模型轻量化以提升工程实用性" |
| **Plan B-3** | R² < 0.80 | 转为**知识图谱推荐**（规则+相似性）而非端到端ML预测 | "从ML预测转向知识驱动的推荐引擎，更符合工业落地需求" |

**Plan B-3具体方案**：
- 已有知识图谱（LightRAG + Neo4j）可支撑基于相似团簇的属性推荐
- 展示"给定目标性能 → 推荐最相似已知团簇"而非"精确预测形成能"
- 答辩角度从"预测精度"转向"数据资产价值释放"

> **CEO关注的核心问题**：PPT工程 vs 真实能力。Plan B确保即使ML效果有限，竞赛展示仍有实质内容。

## 12. 演示场景

### 场景一：多模态数据提取

1. 上传一篇核材料PDF文献
2. 系统自动提取材料名称、成分、物性参数

1. 输入"Zr-4合金在360°C的腐蚀速率"

3. 可视化展示材料关系网络

1. 指定目标：UO₂团簇，优化目标为形成能最小化
2. ML模型预测推荐团簇构型
3. 展示预测精度（R²、MAE vs DFT基准）
