## 第1部分：验证数据现状

### Q1.1: nucpot 中现有的验证数据类型

**已实现自动化验证的 5 个核心性质：**

| 性质 | 英文名 | LAMMPS 方法 | 说明 |
|------|--------|------------|------|
| 晶格常数 | lattice_constant | 能量最小化 | 最基础，几乎所有势函数都能算 |
| 结合能 | cohesive_energy | 能量最小化 | 原子化能量 |
| 弹性常数 | elastic_constants (C11, C12, C44, C33) | 应力-应变法 | 独立弹性常数 |
| 体积模量 | bulk_modulus | 由 Cij 推导 | (C11+2C12)/3 |
| 空位形成能 | vacancy_formation_energy | 缺陷超胞 | 点缺陷性质 |

**已定义在 property-mapping.json 中但尚未全部实现自动验证的性质（12 个）：**

| 性质 | 状态 | 说明 |
|------|------|------|
| 表面能 (surface_energy) | ⚠️ LAMMPS runner 有代码但未全面启用 | 需要构造 slab 模型 |
| 熔点 (melting_point) | ❌ 未实现 | 两相共存法，计算量大 |
| 形成热 (formation_energy) | ❌ 未实现 | 合金体系 |
| 热导率 (thermal_conductivity) | ❌ 未实现 | NEMD/Green-Kubo，计算量大 |
| 间隙形成能 (interstitial_formation_energy) | ❌ 未实现 | 点缺陷扩展 |
| 热膨胀系数 (thermal_expansion) | ❌ 未实现 | NPT 模拟 |
| 比热容 (specific_heat) | ❌ 未实现 | NPT 模拟 |
| 剪切模量 (shear_modulus) | ⚠️ 可从 Cij 推导 | 未单独暴露 |
| 层错能 (stacking_fault_energy) | ❌ 未实现 | 需要特殊超胞构造 |
| 晶界能 (grain_boundary_energy) | ❌ 未实现 | 需要晶界模型 |
| 扩散系数 (diffusion_coefficient) | ❌ 未实现 | MD 轨迹分析 |
| 混合焓 (mixing_enthalpy) | ❌ 未实现 | 合金体系 |

**有参考值的元素/体系（reference_values 表）：**
- ✅ **完整数据（6+ 性质）**：Mo (BCC), Fe (BCC), Zr (HCP), Nb (BCC), U (BCC)
- ⚠️ **部分数据（3-5 性质）**：U (orthorhombic), U-Mo (BCC), U-Zr (BCC), Zr-Nb (HCP)
- ⚠️ **部分数据**：UO₂ (fluorite) — 有晶格+弹性+体模，缺结合能和空位形成能
- ❌ **无参考值**：U-Pu-Zr, UN, UC, SiC, Pu, 所有氧化物（除 UO₂）, 所有液态性质

**验证覆盖率统计（2026-06-10 批量验证后）：**
- 可验证势函数：42/65 (65%)
- 已验证：16/19 新提交验证成功
- 成绩分布：A×1, B×2, C×9, D×3, F×1
- 缺 file_url 无法验证：19 个（Buckingham/Tersoff/AIREBO 等非 EAM 类型）

### Q1.2: 验证数据来源

nucpot 使用**三级缓存查询架构**（Ref-Gap-Fill 系统）：

| 层级 | 数据源 | 说明 |
|------|--------|------|
| **L1** | reference_values 表 (PostgreSQL) | 手动收录的实验/DFT 基准值 |
| **L2** | NFMD parameters 表 (Supabase) | 文献提取的参数数据 |
| **L3A** | llm-wiki 知识库 | 论文摘要中提取的值 |
| **L3B** | ontofuel 本体 | 结构化知识库 |

参考值质量标注：

| source 字段 | 含义 | 可靠性 |
|-------------|------|--------|
| `experimental` | 实验测量 | ⭐⭐⭐⭐⭐ 最高 |
| `DFT` | 密度泛函理论计算 | ⭐⭐⭐⭐ 高 |
| `review` | 综述/手册汇总 | ⭐⭐⭐ 中高 |
| `Smirnov2014` / `Landa2002` | 具体文献引用 | ⭐⭐⭐⭐ 高 |
| `estimated` | 估算值 | ⭐⭐ 低 |
| `calculated` | 从其他值推导（如 (C11+2C12)/3） | ⭐⭐⭐ 中 |

### Q1.3: 验证数据质量

**当前质量评估体系：**

- **分级标准**（LAMMPS runner 实现）：基于相对误差的 A-F 等级
  - A: 相对误差 ≤ 2% — 优秀
  - B: ≤ 5% — 良好
  - C: ≤ 10% — 可接受
  - D: ≤ 20% — 勉强可用
  - F: > 20% — 不可接受

- **不确定性估计**：❌ **当前未实现**。参考值表中没有 uncertainty 列，置信水平也没有量化标注。这是一个明确的差距。

- **数据源的可靠性**：部分参考值标注了 source（如 `experimental`、`DFT`），但混合使用（有的标 `Smirnov2014`，有的标 `review`，有的标 `estimated`），没有统一的质量评分体系。

## 第2部分：定义"验证差距"

### Q2.1: "验证差距"的准确定义

在 nucpot 语境下，"验证差距"应定义为**三层差距的任意组合**：

```
验证差距 (Verification Gap) = Σ(
  势函数覆盖差距,    — 势函数存在但未被验证
  性质覆盖差距,      — 势函数已验证但某些性质缺失
  参考值基准差距      — 性质有计算值但缺少可信参考值进行比较/分级
)
```

具体分类：

1. **势函数未验证 (unverified potential)**：势函数文件存在于数据库中，但 `verifications` 表中没有对应记录（status ≠ completed）
2. **性质未覆盖 (property gap)**：势函数已验证，但仅覆盖部分性质（如只验证了晶格常数，缺弹性常数和空位形成能）
3. **无参考基准 (no reference)**：性质可以由 LAMMPS 计算，但 `reference_values` 表中没有对应的参考值，因此无法给出等级评定（grade = null）
4. **低质量参考 (poor reference)**：有参考值但 source = `estimated` 或数据来源不明确

### Q2.2: 按严重程度分类

| 级别 | 名称 | 标准 | 影响 |
|------|------|------|------|
| **P0 Critical** | 关键安全差距 | 核燃料关键体系（U, UO₂, U-Zr, Zr）的核心性质（晶格常数、弹性常数、空位形成能）未验证 | 可能影响反应堆安全分析中使用该势函数的模拟结果可靠性 |
| **P1 High** | 重要性能差距 | 上述体系缺少次要性质（表面能、扩散系数、混合焒）验证 | 影响辐照损伤、燃料性能建模等领域的模拟质量 |
| **P2 Medium** | 体系覆盖差距 | 非核心但重要体系（U-Pu-Zr, UN, UC, SiC）缺少验证 | 影响先进燃料和事故容错燃料研究 |
| **P3 Low** | 性质扩展差距 | 所有体系缺少扩展性质（热导率、熔点、层错能）验证 | 影响全面材料性能评估 |
| **P4 Info** | 数据质量差距 | 参考值来源标注不完整、缺少不确定性估计、文献 DOI 缺失 | 影响用户对验证结果的信心 |

### Q2.3: 验证优先领域

**第一优先级（P0）：**

核材料安全分析最依赖的势函数体系 × 核心性质：
- U (BCC) — 晶格常数、弹性常数、空位形成能
- UO₂ (fluorite) — 晶格常数、弹性常数、空位形成能
- Zr (HCP) — 晶格常数、弹性常数、空位形成能
- U-Zr (BCC) — 晶格常数、弹性常数
- Fe (BCC) — 晶格常数、弹性常数、空位形成能（钢结构基线）

**缺少验证的后果：**
- 没有验证等级的势函数，研究者无法判断其是否适合特定应用场景
- 安全相关模拟（如燃料包壳辐照损伤）使用未验证势函数，结果可靠性存疑
- 不同势函数之间无法公平比较和选型

## 第3部分：当前手动流程

### Q3.1: 当前如何手动进行验证

当前流程**已部分自动化**，但仍需大量人工介入：

```
1. [手动] 从 NIST IPR / OpenKIM 下载势函数文件
2. [手动] 上传到 Supabase Storage
3. [手动] 在 nucpot 管理后台提交验证请求（选择性质模板）
4. [自动] Vercel BFF 代理 → Cloudflare Tunnel → ThinkStation FastAPI
5. [自动] FastAPI 调度 LAMMPS runner 执行计算
6. [自动] 计算结果与 reference_values 比对，给出 A-F 等级
7. [自动] 结果写回 Supabase verifications 表
8. [手动] 检查验证报告，判断等级是否可接受
9. [手动] 如失败，分析 error_log，修复势函数文件后重新提交
```

### Q3.2: 当前流程中的痛点

| 痛点 | 严重度 | 说明 |
|------|--------|------|
| **势函数文件缺失** | 🔴 高 | 19/65 个势函数缺少 file_url，无法验证。多为 Buckingham/Tersoff/AIREBO 等非 EAM 类型，需要特殊处理 |
| **参考值不完整** | 🔴 高 | 很多材料/结构只有晶格常数参考值，缺弹性常数和空位形成能。没有参考值就无法给 grade |
| **非 EAM 势函数不兼容** | 🟡 中 | Buckingham、Tersoff、AIREBO、ReaxFF 等势函数需要不同的 pair_style，LAMMPS runner 当前主要支持 EAM/MEAM |
| **验证失败后调试困难** | 🟡 中 | error_log 在数据库中，需要人工读取 LAMMPS 日志定位问题（mass 不匹配、缺 pair_coeff 等） |
| **缺少批量操作** | 🟡 中 | 目前逐个提交验证，65 个势函数需要逐个点击或写脚本 |
| **参考值维护混乱** | 🟠 中 | 参考值分两套表（material/structure vs element/crystal_structure），seed data 有重复，source 标注不统一 |
| **无不确定性量化** | 🟠 中 | 参考值没有 uncertainty 列，无法区分高精度实验值和粗略估算值 |
| **缺少 CI/CD 集成** | 🟢 低 | 新增势函数时不会自动触发验证，需要人工提交 |

### Q3.3: 自动化最大的收益

1. **自动验证触发**：新增/更新势函数后自动提交 LAMMPS 验证，无需人工干预
2. **批量验证队列**：一键验证全部未验证势函数，后台 Celery worker 顺序执行
3. **参考值自动填充**（Ref-Gap-Fill 已部分实现）：从 NFMD 数据库和知识库自动查找参考值，减少人工录入
4. **验证差距自动检测**：定期扫描 verifications 表，识别未验证势函数和性质缺口
5. **失败自动重试**：LAMMPS 失败后自动分析错误类型（文件缺失 vs 计算发散 vs pair_style 不兼容），可修复的自动修复
6. **非 EAM 势函数扩展**：自动识别 pair_style 类型，选择正确的 LAMMPS 模板

### Q3.4: 什么不应该自动化

| 需要人工判断的环节 | 原因 |
|---------------------|------|
| **参考值的录入和审核** | 参考值是验证的基准，错误值会污染所有后续验证结果。必须由领域专家审核文献来源和数据质量 |
| **验证结果的最终判断** | A-F 等级基于相对误差，但"可接受"取决于具体应用场景。反应堆安全分析可能要求 B 级以上，而初步筛选 C 级也可以 |
| **非标准势函数的模板选择** | ReaxFF、COMB 等复杂势函数的 LAMMPS 输入需要人工确定 |
| **参考值冲突解决** | 同一性质多个文献给出不同值时（如 U BCC 的弹性常数 Smirnov vs 002_seed），需要专家判断取哪个 |
| **新体系的首次验证** | 新材料体系首次验证时需要人工确认参考值完整性和计算参数合理性 |

## 第4部分：利益相关者需求

### Q4.1: 验证数据的利益相关者

| 利益相关者 | 角色 | 核心需求 |
|------------|------|---------|
| **核材料研究人员** | 势函数使用者 | 快速比较不同势函数的质量，选择最适合研究目标的势函数 |
| **计算材料学家** | 势函数开发者 | 了解自己开发的势函数与实验/DFT 的偏差，指导改进方向 |
| **反应堆工程师** | 燃料性能模拟 | 确认模拟中使用的势函数经过充分验证，结果可信 |
| **安全分析师** | 安全评估 | 验证报告作为模拟结果可靠性的支撑证据 |
| **审评机构** | 监管 | 了解势函数的验证状态和覆盖范围（长期需求，目前不是主要用户） |

### Q4.2: 质量要求

| 利益相关者 | 最低置信水平 | 不确定性容忍度 |
|------------|-------------|--------------|
| 核材料研究者 | C 级（≤10% 偏差）可接受初步筛选 | ±10% 可接受 |
| 反应堆工程师 | B 级（≤5% 偏差）推荐用于工程计算 | ±5% 目标 |
| 安全分析师 | A 级（≤2% 偏差）关键性质必须达到 | ±2% 目标，需附带不确定性 |
| 势函数开发者 | 需要逐性质详细偏差报告 | 需要参考值来源和文献 DOI |

### Q4.3: 覆盖要求

**必须验证的体系（安全相关）：**
- U (α/U), UO₂, Zr (α-Zr), Fe (α-Fe), Zr-Nb — 这 5 个是反应堆核心材料

**应该验证的体系（性能相关）：**
- U-Zr, U-Mo, U-Pu-Zr, Mo, Nb — 燃料和包壳合金

**完整性的最低标准：**
- 每个势函数至少覆盖 3 个核心性质（晶格常数、弹性常数、空位形成能）
- 安全相关势函数的 3 个核心性质至少达到 C 级
- 可接受的差距比例：安全相关 ≤ 10% 未验证；其他 ≤ 30% 未验证

### Q4.4: 报告频率

| 场景 | 频率 | 交付方式 |
|------|------|---------|
| 新势函数提交验证 | 实时（验证完成后） | Web 页面 + API |

## 第5部分：报告格式

### Q5.1: 验证差距报告应包含的内容

```markdown
1. 执行摘要
   - 总势函数数、已验证数、覆盖率百分比
   - 按等级分布（A/B/C/D/F/未验证）
   - 与上次报告的变化趋势

2. 体系级覆盖矩阵
   - 行：元素/体系
   - 列：性质（lattice_constant, C11, C12, C44, E_vac, ...）
   - 单元格：等级（A-F）或 "—"（未验证）或 "?"（无参考值）

3. 关键差距清单
   - 按严重度排序（P0 → P4）
   - 每个差距：势函数、体系、缺失性质、推荐操作

4. 参考值质量报告
   - 有完整文献引用的参考值占比
   - 缺少 uncertainty 的参考值占比
   - 参考值冲突列表

5. 趋势分析（对比上次报告）
   - 新增验证数
   - 等级改善/退化的势函数
```

### Q5.2: 最有用的可视化

| 可视化类型 | 用途 | 优先级 |
|-----------|------|--------|
| **覆盖矩阵热图** | 体系 × 性质，颜色编码等级，一眼看出缺口 | 🔴 最高 |
| **等级分布饼图** | A/B/C/D/F 的数量和占比 | 🟡 高 |
| **体系覆盖率柱状图** | 每个体系的验证完成百分比 | 🟡 高 |
| **差距趋势折线图** | 未验证数随时间的变化 | 🟢 中 |
| **势函数比较雷达图** | 单个势函数的多性质等级一览 | 🟢 中 |

### Q5.3: 需要的下钻功能

1. **体系 → 势函数 → 性质**：
   - 点击热图中的某个体系 → 列出该体系所有势函数
   - 点击某个势函数 → 显示该势函数所有性质的详细验证结果
   - 点击某个性质 → 显示 LAMMPS 计算值 vs 参考值 vs 偏差

2. **差距 → 参考值 → 操作建议**：
   - 点击某个差距 → 显示为何是差距（无参考值？LAMMPS 失败？未提交？）
   - 如果缺参考值 → 显示 Ref-Gap-Fill 查询结果
   - 操作建议（"需要添加参考值"、"需要修复势函数文件"、"需要支持新的 pair_style"）

### Q5.4: 交付机制

| 机制 | 用途 | 优先级 |
|------|------|--------|
| **Web 仪表板** | 实时查看验证状态和差距 | 🔴 必需（管理员后台扩展） |
| **API 端点** | 程序化消费验证数据 | 🔴 必需 |
| **PDF 导出** | 正式报告、存档 | 🟡 重要 |
| **邮件/消息告警** | 验证失败通知 | 🟡 重要（当前未实现） |

## 第6部分：集成点

### Q6.1: 验证与其他 NFM 功能的关系

```
                    ┌─────────────────────────┐
                    │     NucPot (主站)         │
                    │  ┌───────────────────┐  │
  势函数上传 ──────▶│  │  potentials 表      │  │
                    │  └───────┬───────────┘  │
                    │          │               │
                    │  ┌───────▼───────────┐  │
  提交验证 ────────▶│  │  verifications 表   │  │◀──── 验证差距报告
                    │  └───────┬───────────┘  │
                    │          │               │
                    │  ┌───────▼───────────┐  │
                    │  │  reference_values  │  │◀──── 参考值 CRUD（管理员）
                    │  │  (L1 参考基准)     │  │
                    │  └───────┬───────────┘  │
                    └──────────┼──────────────┘
                               │
                    ┌──────────▼──────────┐
                    │  NFMD (Supabase)     │
                    │  ┌───────────────┐  │
                    │  │ parameters 表  │  │◀── L2 参考值来源
                    │  │ literature 表  │  │◀── 文献 DOI 关联
                    │  └───────────────┘  │
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │  LLM-Wiki / Ontofuel │◀── L3 参考值来源
                    └─────────────────────┘

  验证引擎：
  ThinkStation (FastAPI + LAMMPS) ← Cloudflare Tunnel ← Vercel BFF 代理
```

**关键集成关系：**

| 功能 | 关系 | 说明 |
|------|------|------|
| **参考值管理** | 验证的前置依赖 | reference_values 是分级基准，质量直接影响验证结果可信度 |
| **NFMD 参数表** | 参考值补充来源 | Ref-Gap-Fill 系统从 L2 查找缺失的参考值 |
| **文献关联** | 可追溯性 | 验证结果应链接到文献（参考值来源 + 势函数原始论文） |
| **差距填补系统** | 验证差距的解决方案 | 检测到差距后，自动或半自动查找/录入缺失的参考值 |
| **质量门 (Quality Gate)** | 下游消费者 | 势函数在用于正式模拟前应通过质量门（如"3 个核心性质 ≥ C 级"） |

### Q6.2: 应参考的现有验证标准/本体

| 标准/规范 | 适用性 | 说明 |
|-----------|--------|------|
| **OpenKIM Verification Standard** | ⭐⭐⭐⭐ 高 | OpenKIM 的 `check` 系统是业界最成熟的势函数验证框架，定义了标准性质集、参考值来源和差异指标 |
| **NIST IPR 数据格式** | ⭐⭐⭐ 中 | NIST 势函数库的数据结构和验证元数据格式可参考 |
| **MMS (Materials Modeling Standards)** | ⭐⭐⭐ 中 | 欧洲 EMMC 推动的材料建模标准，有验证报告模板 |
| **VASP/Quantum ESPRESSO benchmarks** | ⭐⭐ 低 | DFT 计算的基准数据集，可作为参考值来源补充 |

**最重要的参考：OpenKIM**
- 标准性质集：lattice, elastic, surface, vacancy, stacking-fault, phonon, ...
- 差异指标：相对误差 + 绝对误差
- 参考值来源：实验 + 高精度 DFT，全部标注文献 DOI
- 建议将 nucpot 的验证框架对齐 OpenKIM 的标准

## 总结：四个交付成果

### 1. 验证差距定义与分类框架

```
验证差距 = 势函数未验证 + 性质未覆盖 + 参考值缺失 + 参考值质量不足
分级：P0(安全) → P1(性能) → P2(体系覆盖) → P3(性质扩展) → P4(数据质量)
```

### 2. 利益相关者质量门槛

| 场景 | 最低等级 | 核心性质覆盖率 |
|------|---------|--------------|
| 初步筛选 | C 级 | ≥ 3/5 |
| 工程计算 | B 级 | ≥ 4/5 |
| 安全分析 | A 级 | 5/5 全覆盖 |

### 3. 当前流程痛点和自动化机会

最大收益：(1) 自动验证触发 (2) 批量验证 (3) 参考值自动填充 (4) 差距自动检测
必须保留人工：(1) 参考值审核 (2) 最终验证判断 (3) 非标准势函数模板 (4) 参考值冲突解决

### 4. 报告格式建议

核心视图：体系×性质热图 + 等级分布 + 差距清单
交付方式：Web 仪表板（优先）+ API + PDF 导出
